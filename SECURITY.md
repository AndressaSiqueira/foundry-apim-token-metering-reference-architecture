# Security Policy

## Supported Versions

| Version | Supported |
|---|---|
| main branch | Yes |
| Tagged releases | Latest release |

## Reporting a Vulnerability

**Please do NOT report security vulnerabilities through public GitHub Issues.**

To report a security vulnerability:

1. Open a **private** [GitHub Security Advisory](https://github.com/AndressaSiqueira/foundry-apim-token-metering-reference-architecture/security/advisories/new) in this repository.
2. Provide a description of the vulnerability, steps to reproduce, and
   potential impact.
3. You will receive an acknowledgement within 5 business days and a resolution
   timeline within 10 business days.

## Security Design

For the security architecture of this reference implementation, see
[docs/security.md](docs/security.md).

## Responsible Disclosure

We follow coordinated disclosure. We ask that you:

- Give us reasonable time to address the issue before public disclosure.
- Not exploit the vulnerability beyond what is necessary to demonstrate it.
- Not access, modify, or delete data belonging to others.

## Hall of Fame

We gratefully acknowledge security researchers who report vulnerabilities
responsibly. Researchers will be credited in the release notes (unless they
prefer to remain anonymous).
